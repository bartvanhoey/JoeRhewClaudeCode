# Persona: The Builder / Developer

**Who they are:** Software engineers, technical founders, and AI/platform teams. The core, highest-volume user base — not a fringe segment. Anthropic's own usage data (Economic Index) shows Computer/Mathematical occupations at ~35% of Claude.ai conversations and ~44% of API traffic; coding alone is ~36% of Claude.ai usage.

## Pain points & use cases
- Shipping code faster without sacrificing quality or control over large, unfamiliar codebases.
- Wants autonomy with supervision — agentic help that can read a codebase, edit files, and run commands, but with checkpoints (e.g. "Plan Mode") rather than unchecked autonomy.
- Builds products *on top of* Claude (API/platform), not just chat — this persona doubles as a platform customer, not only an end user.

## Messaging hooks Anthropic uses
- *"Create what's exciting. Maintain what's essential."* — Claude Code tagline.
- *"Reads your codebase, edits files, and runs commands across your terminal, IDE, desktop app, and browser. Build, debug, and ship with natural language."* (https://claude.com/product/claude-code)
- Productivity proof point cited on the Enterprise page: *"Engineers using Claude Code ship 3x more code and merge 31% more pull requests."* (https://www.anthropic.com/product/enterprise)
- Tone: builder/maker, momentum-focused, speed and craft — not risk/governance language.
